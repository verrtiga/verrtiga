## 🚀 Обо мне

QA / frontend-ориентированный специалист, работаю с реальными веб-системами, аналитикой и данными.

Основной фокус:
- тестирование и улучшение веб-приложений
- анализ API и поведения системы
- нагрузочное тестирование
- участие в разработке и улучшении frontend-части
- автоматизация процессов с использованием AI

Работаю с живыми проектами (аналитика обращений): нахожу баги, выявляю узкие места, анализирую нагрузку и помогаю улучшать стабильность и UX системы.

---

## 🧠 QA и тестирование

- Ручное и исследовательское тестирование
- Поиск, воспроизведение и анализ сложных багов
- Оформление баг-репортов (Notion, Jira)
- Приоритизация и структурирование дефектов
- Тестирование фильтров, логики данных и UI
- Проверка граничных кейсов и нестабильных сценариев
- Работа с нестабильным поведением системы (интермиттентные баги)

---

## 🌐 API и системный анализ

- Работа с Chrome DevTools (Network, Performance)
- Анализ запросов и ответов API
- Поиск причин ошибок (504, некорректный JSON и т.д.)
- Анализ polling, нагрузки и поведения системы
- Понимание взаимодействия frontend ↔ backend
- Работа с большими объёмами данных

---

## ⚡ Нагрузочное тестирование

- k6
- Проверка системы под нагрузкой
- Выявление узких мест
- Анализ деградации производительности
- Работа с периодами и большими выборками данных

---

## 💻 Frontend

- HTML, CSS, JavaScript
- SCSS, BEM-архитектура
- Адаптивная и кроссбраузерная верстка
- Работа со структурой интерфейсов
- Оптимизация UI под реальные данные
- Работа с состояниями интерфейса (loading, error, empty)
- Улучшение UX на основе тестирования

📌 Подход к frontend:
- семантическая верстка
- валидный HTML
- доступность (a11y)
- отказ от "костылей"
- масштабируемая архитектура стилей
- мобильный-first подход
